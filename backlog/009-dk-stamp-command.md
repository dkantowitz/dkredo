## Summary

Implement the `dk-stamp` command — records current input state after a
successful build so the next `dk-ifchange` can detect changes.

## Current State

Internal packages implemented. CLI dispatch exists. The `stamp` case is a stub.

## Analysis & Recommendations

Algorithm per `dk-redo-implementation.md`:

```
1. Parse flags (--append, -v, -q, -M <depfile>), extract label and inputs
2. Resolve inputs via resolve.Resolve() — stdin paths spliced at position
3. If -M <depfile>: parse makefile dep format via resolve.ReadDepfile()
   - Strip "target:" prefix, handle backslash-newline continuations
   - Merge extracted paths into resolved input list
4. Hash all resolved files via hasher.HashFile/HashDir
5. If --append: read existing stamp, merge via stamp.Append()
6. Write stamp atomically via stamp.Write()
7. Exit 0 on success, exit 2 on error
```

Exit codes per `dk-redo.md:243-244`:
- `0` — stamp written
- `2` — error

The `--append` flag per `dk-redo.md`:
- New files added to stamp
- Existing files have facts updated
- Files not in current call are preserved

The `-M <depfile>` flag per `dk-redo.md`:
- Reads gcc/clang makefile-format dependency files (as generated by `-MD -MF`)
- Parses `target: dep1 dep2 \`-continuation format
- Extracts dependency paths and merges into input list
- Can combine with positional args, stdin, and `--append`

Non-existent input files per `dk-redo.md`:
record `missing:true` (no hash or size). This enables the bootstrapping pattern.

**All input modes must work and be tested:**
- Positional file arguments
- Directory arguments (hashed recursively)
- `-` (stdin, newline-terminated)
- `-0` (stdin, null-terminated)
- Combinations: `dk-stamp label src/*.c - lib.c` (stdin spliced at position)
- `-M <depfile>`: makefile dep format (gcc `-MD`)
- Combined: `dk-stamp label src/*.c -M .deps/out.d` (args + depfile)
